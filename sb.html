The Study Buddy App
A case study on designing this theoretical app.

Background

Project details.

— The Study Buddy App —

My Approach

I decided to do design this theoretical mobile app to get some more practice with UX and UI Design. To come up with an idea for an app to design, I started with a quick mind map - the same mind map used for creating the after high school app. After picking an idea, I used a five phase design process (outlined below) to bring my app from sketches to final screens.

Tools and Techniques


For this project, I started with personas, user flows, and wireframes. Then I moved on to prototyping (using HTML, CSS, and JavaScript in Adobe Dreamweaver) followed by a user test with my prototype. Lastly, I used Adobe Illustrator to create final screens for the app.

Inspiration


My main inspiration for this project was a need felt by many college students: the desire to have a person to study with. At large universities, there are often large class sizes, but sifting through all of one's classmates to find a person to study with can be difficult. A social app created for aiding in a student's search for a "study buddy" would help to fill this need.

Product Objectives

The purpose of developing this app was simply to connect people that wished to work together for homework, studying, or other coursework.

User Needs

To help anticipate user needs, I created a couple of personas using Adobe Illustrator. These personas acted as realistic representations of fictional users that would potentially visit the site, and helped me consider how to cater my design for potential users.

Phase II: Scope

Meeting needs and objectives.

— The Study Buddy App —

Summary

In this phase of the design process, I like to focus on the the tasks that the product will perform (function specifications) and requirements of the content held by the product.

Function Specifications

The Study Buddy App will:

  • Allow users to post a status visible to all users (similar to FaceBook or Twitter) indicating what they are studying.
  • Allow users to contact each other via study requests, or direct messages.
  • Allow users to add a new course to their school, if unlisted.

    • Content Requirements

    Images. Users will be permitted to upload a single profile photo (1MB max).

    Phase III: Scructure

    User interaction and navigation.

    — The Study Buddy App —

    Summary

    In the Structure Phase, my main focus is interaction design (how the product and the user interact) and information architecture (how the content is organized).

    Interaction Design

    Here, my starting point was to consider using signifiers which provide the user with intuitive cues on what the system will do and how the system will respond. Here's some examples of signifiers that I've included in the design of my portfolio site:

  • A thick border around small blocks of text to give a button-like appearance.
  • A plus sign to indicate adding a person as a contact.
  • Light shadows indicating depth for clickable items.
  • An envelope icon indicating a messaging function.

    • In addition to signifiers, there are many other interaction design elements to consider including error handling, discoverability, feedback, etc.

    Information Architecture

    For this project, I decided to use a hierarchical structure to organize pages and information. To illustrate how a user would navigate my site, I created a user flow.

    Summary

    In this phase, I used my user flow and personas to guide me while creating my wireframes and the prototype for my app. By using wireframes, I was able to create a sort of skeleton upon which more detailed elements can be placed later on. Afterward, I built an app prototype in Adobe Dreamweaver using HTML, CSS, and JavaScript (see footage of prototype in image gallery).

    User Testing

    After creating a prototype I designed a testing plan with the goal of learning about any pain points associated with using the app, and determining if the app's main functions were intuintive.

    From user testing, I learned:

  • The star on the favorites page needed to be more noticeable.
  • The envelope and the plus sign were recognizeable as a clickable elements.
  • The fuctions of the envelope and the plus sign were intuitive.
  • Users did not recognize the profile picture as a clickable element.
  • Users prefered to see shorter names for operations (e.g. "Post a Study Status" was too long).

    • Phase V: Surface

    Visual design and UI elements.

    — The Study Buddy App —

    Summary

    The Surface Phase is where I build ontop of the wireframes and user flows that I built in the earlier phases, while applying the design considerations that I encountered from the results of my user test.

    From user testing, I was able to make the following design changes:

  • The profile image was given more surrounding whitespace to allow it to be perceievd as a clickable item.
  • The names for operations were shortened (e.g. "Post a Study Status" became "Status").
