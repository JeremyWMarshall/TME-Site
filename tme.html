My Approach

To design my portfolio site, I used a design process that is broken down into five phases; Strategy, Scope, Structure, Skeleton, and Surface. Each of the phases are outlined below.

Tools and Techniques


For this project, I used personas, user flows, and wireframes as I molded my ideas. Then I used Adobe Dreamweaver to code the website using HTML, CSS, and JavaScript.

Inspiration


My main inspiration for this project was Sanel Selimovic's personal page as well as the FabricEleven site. I greatly admired their use of flat design and color for the visual appearance their sites.

Phase I: Strategy

Product objectives and user needs.

— The Marshall Experience —

View Personas

Product Objectives

The purpose of having a portfolio site was to:

  • Display/explain design work, design skills, and relevant experience.
  • Provide users with ways to contact me directly.

    • User Needs

    To help anticipate user needs, I created a couple of personas using Adobe Illustrator. These personas acted as realistic representations of fictional users that would potentially visit the site and helped me consider how to cater my design for potential users.

    Phase II: Scope

    Meeting needs and objectives.

    — The Marshall Experience —

    Summary

    In this phase of the design process, I like to focus on the the tasks that the product will perform (function specifications) and requirements of the content held by the product.

    Function Specifications

    The portfolio site will (1) send emails to me on behalf of site visitors via a contact page and (2) have files available for download, including my resume and some portfolio elements.

    Content Requirements

    Text. Short descriptions (50-100 words per description) will be used to reduce information overload for the user.

    Images. Images would be viewable on the site in a gallery. For downloadable images, small file sizes (2MB max) will be used.

    Videos. Videos will be hosted on youtube, but playable on the portfolio site in a video gallery.

    Standalone Content. For downloadable files (portfolio, resume) will also have small file sizes (≤200Kb).

    Phase III: Scructure

    User interaction and navigation.

    — The Marshall Experience —

    Summary

    In the Structure Phase, my main focus is interaction design (how the product and the user interact) and information architecture (how the content is organized).

    Interaction Design

    Here, my starting was to consider using signifiers which provide the user with intuitive cues on what the system will do and how the system will respond. Here's some examples of signifiers that I've included in the design of my portfolio site:

  • An arrow on the first page indicating the location of more information.
  • A play icon ontop of images representing playable videos.
  • Light shadows indicating depth for clickable items.
  • Red highlighting on the navigation indicating the user's current location.

    • In addition to signifiers, there are many other interaction design elements to consider including error handling, discoverability, feedback, etc.

    Information Architecture

    For this project, I decided to use a hierarchical structure to organize pages and information. To illustrate how a user would navigate my site, I created a user flow.

    In this phase, I used my user flow and personas to guide me while creating my wireframes for my website. By using wireframes, I was able to create a sort of skeleton upon which more detailed elements can be placed later on.
